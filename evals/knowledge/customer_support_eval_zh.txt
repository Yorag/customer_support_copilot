Agentia 客服知识库（中文评测版）

一、产品定位
Agentia 是一个面向 AI 自动化团队和内部平台团队的工作流编排与智能代理平台。
系统支持知识检索、工具调用、版本管理、Webhook 回调、批量导入和审计日志查询等能力。

二、身份认证与单点登录
1. 专业版及以上版本支持基于 SAML 2.0 的单点登录。
2. 管理员可在“控制台 -> 安全 -> 单点登录”中开启 SSO。
3. 配置步骤包括：填写 IdP 登录 URL、上传 IdP 元数据证书、设置 NameID 映射为邮箱、保存后执行测试登录。
4. 系统支持在测试通过后启用“强制 SSO 登录”。
5. 暂不支持 tenant 级 dry run 激活模拟。

三、API 与速率限制
1. 工作区级 API 令牌默认速率限制为每分钟 600 次请求。
2. 当超过限制时，接口会返回 `429 Too Many Requests`。
3. 响应头会包含 `Retry-After`，客户端应遵循该值进行重试。
4. 推荐退避策略为 1 秒、2 秒、4 秒、8 秒，最多重试 5 次。
5. 批量写入接口建议使用幂等键，避免重试造成重复写入。

四、Webhook
1. Webhook 请求使用 `X-Agentia-Signature` 头传递签名。
2. 签名算法为 HMAC-SHA256，签名原文为原始请求体。
3. 服务端接收超时时间为 5 秒。
4. 失败重试策略为 1 分钟、5 分钟、20 分钟、60 分钟、180 分钟，共最多 5 次。
5. Webhook 事件可能重复投递，接收方应使用事件 ID 做幂等处理。

五、导入任务
1. CSV 导入要求 UTF-8 编码。
2. 单次导入最多支持 100000 行。
3. 系统会为失败行生成错误报告，报告中包含行号和字段级错误原因。
4. 常见失败原因包括：邮箱格式无效、必填列缺失、枚举值不合法。
5. 导入完成前，任务状态可能停留在 `queued`、`running`、`validating`、`finalizing`。

六、审计日志与导出
1. 审计日志默认保留 180 天。
2. 支持导出最近 90 天的审计日志文件。
3. 当前不支持自定义审计导出保留期覆盖。
4. 当前不支持按租户级别配置独立的审计导出生命周期策略。

七、SCIM 与用户同步
1. 平台支持标准 SCIM 用户预配。
2. 当前不支持自定义区域级故障切换 SCIM bridge。
3. 如用户同步失败，建议先核对映射字段、目录属性、最近一次同步错误和租户标识。

八、计费与政策边界
1. 重复扣费、争议扣费、退款、补偿、SLA 抵扣均需要人工审核。
2. 客服自动流程不得承诺退款、补偿、信用额度或合同例外。
3. 取消订阅会在当前计费周期结束后生效。
4. 已开具发票如需更正，需要人工复核。
5. 涉及合同条款、法务审阅、SLA 赔付、重大争议的工单必须升级人工处理。

九、支持边界
1. 客服流程只处理产品使用、技术问题、计费政策、反馈收集等范围内请求。
2. 不处理 SEO 推广、外链合作、餐厅预订、招聘推销、赞助投放等非客服事务。

十、回答约束
1. 如果知识库没有明确说明某项能力，则应明确表示“不确定”或“当前知识中没有该信息”。
2. 不得虚构未记录的企业特性、合同承诺或退款结论。
3. 对于高风险政策问题，应明确说明需要人工审核。
