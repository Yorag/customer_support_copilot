Agentia 产品与支持知识库（中文正式版）

文档说明
本文件是 Agentia 客服自动化系统使用的正式知识源，面向客户支持、解决方案顾问、售前实施和内部运营团队。
文档目标是提供统一、可引用的产品事实、能力边界、套餐差异、配置说明、运维约束和商业政策摘要。
本文档不是模型提示词，也不是人工工单处理 SOP。它描述的是产品本身和公开支持边界。

================================================================================
1. 产品概述
================================================================================

产品类型
Agentia 是一款面向企业内部平台团队、数字化运营团队和 AI 自动化团队的企业级 SaaS 产品。
它的核心定位是“工作流编排与智能代理治理平台”，用于把知识检索、工具调用、审批流程、Webhook 回调、批量数据操作和审计留痕整合到统一的工作台中。

典型使用场景
- 为内部员工门户、客户支持台和运营后台配置自动化流程
- 为业务系统接入工作流、Webhook 和 API 集成
- 为 AI Agent 提供受控的知识访问、工具执行和版本发布能力
- 为管理员提供审计日志、导入导出和权限治理能力

部署方式
- Agentia 标准版本以云服务方式提供
- 默认不支持客户自托管部署
- 如需私有化或专属部署，需要单独的企业合同和交付评估

主要能力域
- 身份认证与单点登录
- SCIM 用户预配与同步
- 工作区级 API
- Webhook 回调
- 批量导入与任务执行
- 审计日志与导出
- 版本管理与自动化运行

支持范围
Agentia 客服支持主要覆盖以下问题：
- 产品功能介绍
- 配置说明
- 已发布能力的使用方式
- 常见技术问题排查方向
- 计费政策说明
- 反馈收集与升级分流

================================================================================
2. 套餐与版本边界
================================================================================

基础版
- 适合小规模团队试用和轻量工作流场景
- 不提供 SAML 单点登录
- 不提供 SCIM 用户预配
- 提供基础工作流、基础日志查看和标准 API 能力

专业版
- 面向需要更完整身份治理和系统集成的团队
- 支持基于 SAML 2.0 的单点登录
- 不提供 SCIM 用户预配
- 提供工作区级 API、Webhook 和较完整的管理控制台能力

企业版
- 面向需要集中治理、身份集成和审计能力的大型组织
- 支持基于 SAML 2.0 的单点登录
- 支持标准 SCIM 用户预配
- 支持更完整的导出、审计与商业支持流程

当前不支持的能力
- tenant 级 dry run 激活模拟
- 自定义区域级故障切换 SCIM bridge
- 按租户单独覆盖审计导出保留期
- 未经企业合同约定的私有化交付

================================================================================
3. 身份认证与单点登录
================================================================================

支持情况
- 专业版及以上版本支持基于 SAML 2.0 的单点登录
- 企业版支持更完整的身份治理场景，但基础 SAML 启用方式与专业版一致

配置入口
管理员可以在“控制台 -> 安全 -> 单点登录”中完成 SSO 配置。

标准配置项
- IdP 登录 URL
- IdP 元数据或签名证书
- Issuer 或实体标识
- NameID 映射
- 测试登录开关

推荐配置流程
1. 填写或导入 IdP 提供的 SAML 配置
2. 将 NameID 映射为邮箱标识
3. 保存配置后执行测试登录
4. 至少确认一个管理员账户可以正常登录后，再启用强制 SSO

常见限制
- 当前不支持 tenant 级 dry run 激活模拟
- 标准产品形态下不支持多租户共享一个可编辑的 SSO 模板

SSO 登录失败时常见原因
- IdP 登录 URL 配置错误
- 元数据证书过期或轮换后未同步更新
- Issuer 或 Entity ID 不匹配
- NameID 映射不是邮箱
- 回调或断言配置不一致
- 浏览器缓存或时钟偏差导致登录断言校验失败

如果客户已经提供的关键信息包括
- 浏览器版本
- 工作区标识
- 错误码
- 复现时间
- 最近是否变更过证书或配置

则后续支持应围绕这些事实继续定位，而不是重复回到最基础的环境采集。

================================================================================
4. SCIM 用户预配与同步
================================================================================

支持情况
- 企业版支持标准 SCIM 用户预配
- 当前不支持自定义区域级故障切换 SCIM bridge
- 当前不支持非标准扩展协议的专有预配桥接

标准使用方式
- 为工作区创建专用的 SCIM 接入配置
- 在身份源中配置 SCIM endpoint 与 token
- 通过标准属性映射同步用户和组信息

常见同步异常来源
- SCIM token 已更新但身份源侧未同步
- 属性映射变更后出现字段不兼容
- 身份源发送了不符合当前目录结构的字段
- 工作区标识或租户目标配置错误

支持排查时最有价值的信息
- 工作区或租户标识
- 最近一次成功同步时间
- 最近一次失败时间
- 身份源返回的错误内容
- 是否更换过 token 或修改过映射

5. API 能力与速率限制
================================================================================

API 范围
Agentia 提供工作区级 API，用于：
- 工作流触发
- 任务查询
- 数据写入
- 管理动作
- 集成状态同步

速率限制
- 工作区级 API token 默认速率限制为每分钟 600 次请求
- 超过限制时，接口返回 `429 Too Many Requests`
- 响应头包含 `Retry-After`

重试建议
- 客户端优先遵循 `Retry-After`
- 如未返回更严格的等待值，可使用指数退避
- 参考退避序列：1 秒、2 秒、4 秒、8 秒
- 建议最多重试 5 次

集成建议
- 对于批量写入或可能发生重放的调用，建议使用幂等键
- 对于高频调用，建议在调用侧实现请求排队或速率平滑

当前未公开承诺的能力
- 未公开单独提供 tenant 级自定义限流策略
- 未公开提供按单个 API key 定制的实时限速面板

6. Webhook
================================================================================

Webhook 概述
Agentia 支持把事件变更推送到客户系统或中间件，用于流程状态同步、任务通知和集成联动。

签名机制
- Webhook 请求使用 `X-Agentia-Signature` 头传递签名
- 签名算法为 HMAC-SHA256
- 签名基于原始请求体计算

交付超时
- 服务端接收超时时间为 5 秒
- 若接收端在规定时间内没有返回成功状态，当前投递会被视为失败

失败重试策略
- 第 1 次重试：1 分钟后
- 第 2 次重试：5 分钟后
- 第 3 次重试：20 分钟后
- 第 4 次重试：60 分钟后
- 第 5 次重试：180 分钟后

实现建议
- 接收端应基于事件 ID 做幂等处理
- 接收端应保留原始请求体用于验签
- 接收端应避免在同步路径中执行耗时过长的业务逻辑

常见问题
- 密钥轮换后仍使用旧密钥验签
- 验签时使用了格式化后的 JSON，而不是原始请求体
- 接收端超时或返回了非 2xx 状态码

7. 批量导入
================================================================================

导入格式
- CSV 文件要求使用 UTF-8 编码
- 单次导入最多支持 100000 行

典型导入流程
1. 上传文件
2. 进入排队状态
3. 执行校验
4. 执行写入
5. 生成结果与失败报告

常见任务状态
- `queued`
- `running`
- `validating`
- `finalizing`
- `completed`
- `failed`

常见失败原因
- 邮箱格式无效
- 必填列缺失
- 枚举值不合法
- 编码不正确
- 文件列结构与当前模板不匹配

错误报告
- 失败任务通常会生成错误报告
- 错误报告通常包含行号和字段级错误原因

如果客户反馈导入任务长时间停留在 `finalizing`
最需要确认的事实包括：
- 任务 ID
- 文件行数规模
- 是否生成错误报告
- 最近一次复现时间
- 是否在最近更改过模板或字段映射

================================================================================
8. 审计日志与导出
================================================================================

审计日志保留期
- 审计日志默认保留 180 天

导出范围
- 支持导出最近 90 天的审计日志文件

当前限制
- 当前不支持自定义审计导出保留期覆盖
- 当前不支持按租户级别配置独立的审计导出生命周期策略

适用场景
- 管理审计
- 内部安全排查
- 合规留痕
- 关键操作回溯

9. 计费、退款与商业政策摘要
================================================================================

计费政策范围
客服可以解释：
- 基本计费规则
- 取消订阅的生效时间
- 发票更正是否需要人工复核
- 商业问题进入人工审核的边界

取消订阅
- 取消订阅会在当前计费周期结束后生效
- 标准流程下不会因为提交取消请求而立即终止已生效的服务周期

发票更正
- 已开具发票如需更正，需要人工复核

重复扣费与退款
- 重复扣费、争议扣费和退款请求都需要人工审核
- 自动化客服流程不得直接承诺退款结果
- 人工审核通常会核对工作区标识、账单或发票 ID、扣费日期、金额和问题描述

SLA、补偿与争议
- 涉及 SLA 抵扣、补偿、法务审阅、重大争议或合同例外的请求必须进入人工审核
- 自动化客服流程不得直接承诺补偿、信用额度或合同例外

10. 反馈、建议与非客服范围
================================================================================

产品反馈
- 客户提出功能建议、体验意见或流程改进建议时，可进入反馈收集流程
- 标准反馈处理不代表产品已承诺上线时间

非客服范围
以下内容不属于 Agentia 标准客服范围：
- SEO 推广
- 外链合作
- 赞助投放
- 招聘推销
- 与产品支持无关的商务拓展

================================================================================
11. 已公开能力边界摘要
================================================================================

以下能力在当前公开知识中明确不支持或未公开支持：
- tenant 级 dry run 激活
- 自定义区域级故障切换 SCIM bridge
- 按租户覆盖审计导出保留期
- 未经合同约定的私有化部署
- 自动承诺退款或补偿

================================================================================
12. 使用本知识库时的事实判断原则
================================================================================

1. 如果某项能力在本文档中没有被明确说明，则不应把它表述成已正式支持。
2. 如果某项商业结果需要人工审核，则不能把它表达为已经批准。
3. 如果客户问题已经提供了充分上下文，回复应在既有事实上继续推进，而不是回退到泛泛的信息收集。
4. 本文档优先提供产品事实和正式边界；具体回复风格、语气和风险控制规则应由客服流程或策略模块补充。
